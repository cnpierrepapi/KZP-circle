# Agent Rewards — build & test

This is a complete, build-ready Anchor project. The Rust/Solana/Anchor toolchain is **not**
installed on this machine yet, so the steps below install it, then build and test.

## 0. Use WSL2 (strongly recommended on Windows)
Native-Windows Anchor builds are painful. Install WSL2 + Ubuntu and do everything inside it:
```powershell
wsl --install -d Ubuntu
```
Then open Ubuntu and run the rest there. Put the project on the Linux side
(e.g. `cp -r /mnt/c/Users/HP/agent-rewards ~/agent-rewards`) for sane build speed.

## 1. Toolchain
```bash
# Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
source "$HOME/.cargo/env"

# Solana CLI
sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"

# Anchor via avm
cargo install --git https://github.com/coral-xyz/anchor avm --force
avm install 0.30.1
avm use 0.30.1

# Node deps for the tests
npm install
```

## 2. Generate the real program id (replaces the placeholder)
```bash
solana-keygen new            # if you have no wallet yet
anchor build                 # first build creates target/deploy/agent_rewards-keypair.json
anchor keys sync             # rewrites declare_id! in lib.rs AND the ids in Anchor.toml
anchor build                 # rebuild with the correct id
```

## 3. Test (spins up a local validator automatically)
```bash
anchor test
```
Expected: 8 passing tests — vault init, deposit, the three work-type payouts, and the three
rejections (replayed nonce, untrusted oracle, reward > vault balance), then owner withdrawal.

## 4. Deploy to devnet (only when tests are green — ask before doing this)
```bash
solana config set --url devnet
solana airdrop 2
anchor deploy --provider.cluster devnet
```

## Notes
- `target/types/agent_rewards.ts` (imported by the test) is generated by `anchor build`. The
  test won't type-check until you've built once.
- The placeholder program id `1111...1111` only exists so the file parses; `anchor keys sync`
  replaces it with your generated keypair. Do not deploy with the placeholder.
- Devnet only. Unaudited. The oracle is trusted — see `../Downloads/agent-rewards-mock/README.md`.
